# PRD: 대학생 인턴/신입 공고 탐색 지원 MCP

## 1. Product Summary

InternMate MCP는 대학생과 취업 준비생이 자연어로 인턴/신입 공고를 찾을 수 있도록 돕는 MCP 서버이다. 사용자가 "IT 기업 인턴 공고 찾아줘"처럼 질문하면 사람인 채용정보 API를 조회하고, 기본적으로 오늘부터 14일 이내 마감되는 인턴/신입/경력무관 공고를 추천순으로 정리한다.

### One-line Pitch

사람인 채용정보 API를 기반으로 대학생이 지금 지원할 만한 인턴/신입 공고를 자연어로 찾아주고 추천 이유까지 정리하는 MCP 서버.

### Working Name

- InternMate MCP
- EntryJob Radar MCP
- 대학생 인턴 레이더
- 지원각 MCP

본 문서에서는 임시 제품명을 `InternMate MCP`로 사용한다.

## 2. Background

MCP 테스트 환경에서는 사용자가 긴 정보를 붙여넣기보다 자연어 질문만 입력한다. 따라서 사용자가 공고 데이터를 직접 제공하는 방식은 적합하지 않다.

대학생은 인턴, 신입, 체험형 공고를 자주 찾지만 채용 사이트 검색 조건은 복잡하다. 같은 "IT 인턴" 요청이라도 사용자는 마감이 임박한 공고, 대학생도 지원 가능한 공고, 경력 조건이 낮은 공고, 지역이 맞는 공고를 한 번에 보고 싶어 한다.

## 3. Problem Definition

대학생에게 필요한 것은 단순 채용 검색 결과 목록이 아니라 "지금 내가 지원해볼 만한 공고"이다.

### User Pain Points

1. 채용 사이트 검색 조건을 직접 조합하기 번거롭다.
2. 인턴, 신입, 경력무관 공고를 분리해서 찾기 어렵다.
3. 마감이 너무 먼 공고보다 지금 준비해야 할 공고를 우선 보고 싶다.
4. 공고 제목만 보고는 대학생 지원 가능성이 높은지 알기 어렵다.
5. 여러 공고를 비교할 때 추천 이유와 준비 포인트가 필요하다.

### Core Problem Statement

대학생이 자연어로 말한 직무, 지역, 조건을 기반으로 신뢰 가능한 채용 API에서 인턴/신입 공고를 검색하고, 14일 이내 마감 공고를 추천순으로 요약해야 한다.

## 4. Target Users

### Primary Users

- 인턴을 찾는 대학생
- 신입 공고를 찾는 졸업 예정자
- 경력무관 공고를 찾는 취업 준비생

### Secondary Users

- 전공과 맞는 직무를 탐색하는 저학년 대학생
- 취업 동아리/스터디 운영자
- 공고를 빠르게 비교하고 싶은 멘토 또는 조교

## 5. Data Source

### Primary Source: 사람인 채용정보 API

사람인 채용정보 API를 1차 데이터 소스로 사용한다.

이유:

- 채용 공고 전문 데이터이므로 일반 웹 검색보다 신뢰도가 높다.
- 인턴/신입/경력무관 공고 탐색에 직접 연결된다.
- 회사명, 공고명, 지역, 경력, 학력, 마감일, URL 등 추천에 필요한 핵심 정보가 구조화되어 있다.
- 공고 ID 기반 상세 조회 흐름을 만들 수 있다.

### Source Limitations

- 사람인 API는 채용정보 API이므로 공모전/대외활동은 MVP 범위에서 제외한다.
- API 사용에는 access key 신청과 승인이 필요하다.
- API 인터페이스가 베타이거나 변경될 수 있으므로 구현 시 문서 기준으로 최소 필드에 의존한다.
- API 장애 또는 키 미설정 상황에서는 명확한 오류를 반환한다.

## 6. Product Goals

### Goals

1. 자연어 요청에서 직무, 키워드, 지역, 고용 형태, 마감 범위를 구조화한다.
2. 사람인 API로 인턴/신입/경력무관 공고를 검색한다.
3. 기본적으로 오늘부터 14일 이내 마감 공고를 우선 추천한다.
4. 대학생 관점의 추천 이유를 제공한다.
5. 공고 상세 정보를 카카오톡에서 읽기 좋은 요약으로 제공한다.
6. 선택한 공고에 대한 지원 준비 체크리스트를 만든다.

### Non-goals

1. 공모전, 대외활동, 서포터즈 검색은 MVP 범위가 아니다.
2. 사용자의 이력서나 개인정보를 저장하지 않는다.
3. 합격 가능성을 예측하지 않는다.
4. 사람인 외 채용 사이트를 직접 크롤링하지 않는다.
5. 채용 지원을 대신 제출하지 않는다.

## 7. Key User Scenarios

### Scenario 1: IT 인턴 공고 추천

User:

```text
IT 기업 인턴 공고 찾아줘.
```

Expected behavior:

- 키워드: IT, 개발, 데이터, 인턴
- 마감 범위: 오늘부터 14일 이내
- 경력 조건: 인턴, 신입, 경력무관 우선
- 결과: 추천순 5개

Expected answer:

```text
오늘 기준 2주 안에 마감되는 IT 인턴/신입 공고를 추천드릴게요.

1. ABC테크 - 백엔드 개발 인턴
- 마감: 2026.07.12
- 지역: 서울 강남구
- 경력: 신입/경력무관
- 추천 이유: IT 직무 키워드와 일치하고 마감까지 준비 시간이 있습니다.
- 링크: https://...
```

### Scenario 2: 지역 조건 포함 검색

User:

```text
서울에서 지원 가능한 마케팅 신입 공고 추천해줘.
```

Expected behavior:

- 키워드: 마케팅
- 지역: 서울
- 공고 유형: 신입 또는 경력무관
- 마감 범위: 기본 14일

### Scenario 3: 상세 정보 확인

User:

```text
1번 공고 자세히 알려줘.
```

Expected behavior:

- 이전 검색 결과의 첫 번째 공고 ID 또는 URL을 사용한다.
- 공고 상세 정보를 조회한다.
- 주요 업무, 자격 요건, 우대사항, 근무지, 마감일, 링크를 요약한다.

### Scenario 4: 지원 준비 체크리스트

User:

```text
이 공고 지원하려면 뭐 준비해야 해?
```

Expected behavior:

- 선택한 공고 상세 정보를 바탕으로 지원 준비물을 추출한다.
- 명시되지 않은 항목은 일반적인 인턴/신입 지원 기준으로 보조한다.
- 이력서, 자기소개서, 포트폴리오, 프로젝트 경험 정리, 마감 리마인드를 제공한다.

## 8. MVP Scope

### Must-have

1. `search_entry_jobs`: 자연어 조건 기반 인턴/신입 공고 검색
2. `get_job_detail`: 특정 공고 상세 조회
3. 사람인 API key 환경 변수 사용
4. 기본 마감 범위 14일
5. 카카오톡에서 읽기 좋은 텍스트 응답
6. API 오류/키 누락 시 명확한 오류 메시지

### Nice-to-have

1. `make_application_brief`: 지원 준비 요약 생성
2. 사용자 전공/관심 직무 기반 추천 사유 강화
3. 지역/직무 코드 매핑 고도화
4. API 실패 시 데모용 샘플 fixture 사용

### Future Expansion

1. 워크넷 등 공공 채용 API 추가
2. 공고 북마크/비교
3. 캘린더 마감 리마인드
4. 이력서/포트폴리오 기반 맞춤 추천
5. 공모전/대외활동 별도 신뢰 소스 확보 후 확장

## 9. Core MCP Tools

### `search_entry_jobs`

자연어 조건을 구조화해 사람인 API에서 대학생 대상 인턴/신입 공고를 검색하고 추천순으로 반환한다.

### `get_job_detail`

검색 결과에서 선택한 공고의 상세 정보를 조회하고 지원 판단에 필요한 핵심 정보를 요약한다.

### `make_application_brief`

선택한 공고에 지원하기 위해 준비해야 할 항목과 마감 전 행동을 체크리스트로 정리한다.

MVP 최소 구현은 `search_entry_jobs`, `get_job_detail` 2개이다. 시간이 가능하면 `make_application_brief`를 추가한다.

## 10. Ranking Criteria

추천순은 MVP에서 단순 점수 기반으로 계산한다.

| Criterion | Score | Description |
|---|---:|---|
| Keyword match | +3 | 사용자 직무/키워드가 제목 또는 직무와 일치 |
| Entry fit | +3 | 인턴, 신입, 경력무관 조건 |
| Deadline within 14 days | +2 | 기본 마감 범위 안에 있음 |
| Not too urgent | +1 | 마감까지 3일 이상 남음 |
| Location match | +2 | 사용자 지역 조건과 일치 |
| Student-friendly education | +1 | 대졸 예정/무관 등 대학생 친화 조건 |

정렬 우선순위:

1. 직무 적합도
2. 대학생 지원 가능성
3. 마감일
4. 지역 조건

## 11. Chat-based Execution Model

사용자는 JSON을 직접 입력하지 않는다. 사용자는 채팅창에 자연어로 질문한다.

```text
사용자:
IT 기업 인턴 공고 찾아줘.

AI 내부 tool 호출:
search_entry_jobs

최종 답변:
오늘 기준 2주 안에 마감되는 IT 인턴/신입 공고를 추천드릴게요.
...
```

MCP tool의 input schema는 AI와 MCP 서버 사이의 내부 `tools/call` arguments 형식이다.

## 12. Privacy and Safety

- 사용자의 개인정보를 요구하지 않는다.
- 사용자의 이력서, 학번, 전화번호, 이메일을 저장하지 않는다.
- 공고 추천은 정보 제공이며 합격 가능성 보장을 하지 않는다.
- 마감일과 상세 조건은 API 기준이므로 최종 지원 전 원문 링크 확인을 안내한다.
- API key는 `.env`에 저장하고 코드에 하드코딩하지 않는다.

## 13. Acceptance Criteria

MVP 완료 기준:

1. `search_entry_jobs` tool이 자연어 조건 기반 검색 입력을 받을 수 있다.
2. 기본값으로 14일 이내 마감 공고를 추천한다.
3. `get_job_detail` tool이 공고 ID 또는 URL 기반 상세 요약을 반환한다.
4. 응답은 MCP `content` 배열의 텍스트로 반환된다.
5. 사람인 API key가 없을 때 명확한 오류를 반환한다.
6. 실제 공고 데이터를 저장하지 않는다.
7. 테스트는 외부 API mock 또는 합성 fixture를 사용한다.

## 14. Open Questions

1. 사람인 API 승인 전까지 데모용 fixture fallback을 넣을지 결정해야 한다.
2. 사람인 직무/지역 코드 매핑을 어느 수준까지 구현할지 결정해야 한다.
3. "대학생 추천" 기준에 학력 조건을 얼마나 강하게 반영할지 결정해야 한다.
4. 공고 상세 조회를 API ID 중심으로 할지 URL 중심으로 할지 확인해야 한다.
